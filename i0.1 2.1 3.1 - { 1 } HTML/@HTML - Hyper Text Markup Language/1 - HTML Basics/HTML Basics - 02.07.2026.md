# > 1.1 What is HTML?

HTML (HyperText Markup Language) হলো একটি **Markup Language**, যা ওয়েব পেজ (Web Page) তৈরির জন্য ব্যবহৃত হয়। এটি কোনো Programming Language নয়; বরং এটি বিভিন্ন **Tag (ট্যাগ)** ব্যবহার করে একটি ওয়েব পেজের কাঠামো (Structure) তৈরি করে।

সহজ ভাষায়, HTML হলো একটি ওয়েবসাইটের **কঙ্কাল (Skeleton)**। যেমন একটি বাড়ি নির্মাণের আগে তার মূল কাঠামো তৈরি করা হয়, ঠিক তেমনি একটি ওয়েবসাইট তৈরির প্রথম ধাপ হলো HTML দিয়ে তার গঠন তৈরি করা।

HTML-এর মাধ্যমে আমরা একটি ওয়েব পেজে বিভিন্ন ধরনের উপাদান (Element) যোগ করতে পারি, যেমন—

* Heading (শিরোনাম)
* Paragraph (অনুচ্ছেদ)
* Image (ছবি)
* Link (লিংক)
* Button (বাটন)
* Table (টেবিল)
* List (তালিকা)
* Form (ফর্ম)
* Audio ও Video
* এবং আরও অনেক কিছু।

HTML একটি **Tag-based Language**। প্রতিটি উপাদান সাধারণত একটি Opening Tag (`<tag>`) এবং একটি Closing Tag (`</tag>`) এর মাধ্যমে লেখা হয়। ব্রাউজার এই ট্যাগগুলো পড়ে ব্যবহারকারীর সামনে একটি সুন্দর ও গঠনবদ্ধ ওয়েব পেজ প্রদর্শন করে।

## HTML-এর পূর্ণরূপ (Full Form)

**HTML = HyperText Markup Language**

* **HyperText** → একটি ডকুমেন্ট থেকে অন্য ডকুমেন্টে Link-এর মাধ্যমে সংযোগ স্থাপন করার পদ্ধতি।
* **Markup** → Tag ব্যবহার করে কনটেন্টের গঠন ও অর্থ নির্ধারণ করার প্রক্রিয়া।
* **Language** → নির্দিষ্ট নিয়ম (Syntax) অনুসরণ করে লেখা একটি ভাষা।

## HTML-এর প্রধান বৈশিষ্ট্য

1. এটি একটি Markup Language।
2. এটি Web Page-এর Structure তৈরি করে।
3. এটি Tag এবং Element-এর মাধ্যমে কাজ করে।
4. এটি শেখা সহজ।
5. এটি সকল আধুনিক Web Browser সমর্থন করে।
6. এটি CSS এবং JavaScript-এর সাথে একসাথে ব্যবহার করা হয়।
7. এটি Cross-Platform অর্থাৎ Windows, Linux এবং macOS-এ একইভাবে কাজ করে।
8. এটি Web Development-এর ভিত্তি (Foundation)।

## HTML কী করতে পারে?

HTML ব্যবহার করে আপনি—

* ওয়েব পেজ তৈরি করতে পারবেন।
* টেক্সট প্রদর্শন করতে পারবেন।
* ছবি যোগ করতে পারবেন।
* অন্য পেজে লিংক তৈরি করতে পারবেন।
* তালিকা (List) তৈরি করতে পারবেন।
* টেবিল তৈরি করতে পারবেন।
* ফর্ম (Form) তৈরি করতে পারবেন।
* Audio ও Video যুক্ত করতে পারবেন।
* সম্পূর্ণ ওয়েবসাইটের কাঠামো তৈরি করতে পারবেন।

## HTML কী করতে পারে না?

HTML একা নিচের কাজগুলো করতে পারে না—

* ওয়েব পেজকে সুন্দরভাবে ডিজাইন করতে পারে না (এর জন্য CSS ব্যবহার করা হয়)।
* Animation তৈরি করতে পারে না (CSS বা JavaScript প্রয়োজন)।
* User-এর সাথে Interaction পরিচালনা করতে পারে না (JavaScript প্রয়োজন)।
* Database পরিচালনা করতে পারে না।
* Server-side Processing করতে পারে না।

## বাস্তব উদাহরণ

ধরুন আপনি একটি বাড়ি তৈরি করছেন।

* HTML → বাড়ির কাঠামো (দেয়াল, দরজা, জানালা)
* CSS → বাড়ির রং, ডিজাইন ও সাজসজ্জা
* JavaScript → বাড়ির বৈদ্যুতিক ব্যবস্থা, অটোমেশন ও স্মার্ট ফিচার

অর্থাৎ—

**HTML = Structure**
**CSS = Design**
**JavaScript = Functionality**

## সারসংক্ষেপ

HTML হলো Web Development-এর সবচেয়ে গুরুত্বপূর্ণ এবং প্রথম প্রযুক্তি। এটি একটি ওয়েব পেজের সম্পূর্ণ কাঠামো তৈরি করে। HTML ছাড়া কোনো ওয়েবসাইট তৈরি করা সম্ভব নয়। আধুনিক Web Development-এ HTML, CSS এবং JavaScript একসাথে ব্যবহার করে একটি সম্পূর্ণ ওয়েবসাইট তৈরি করা হয়।






# > 1.2 HyperText কী?

HyperText হলো এমন একটি Text বা Document, যার মধ্যে **Hyperlink (লিংক)** থাকে। এই লিংকের মাধ্যমে একটি Document থেকে অন্য একটি Document, Web Page, Website বা নির্দিষ্ট কোনো অংশে খুব সহজেই যাওয়া যায়।

সাধারণ Text শুধু পড়ার জন্য ব্যবহৃত হয়, কিন্তু **HyperText** ব্যবহারকারীকে এক জায়গা থেকে অন্য জায়গায় দ্রুত নিয়ে যেতে পারে। এই কারণেই এর নাম **HyperText**, অর্থাৎ এমন Text যা সাধারণ Text-এর চেয়ে বেশি ক্ষমতাসম্পন্ন।

Web Browser-এ আমরা যখন কোনো Link-এ ক্লিক করি এবং অন্য একটি Page-এ চলে যাই, তখন সেটি HyperText-এর মাধ্যমেই সম্ভব হয়।

## HyperText শব্দের অর্থ

* **Hyper** → সাধারণের চেয়ে বেশি বা উন্নত (Beyond Normal)
* **Text** → লেখা বা তথ্য

অর্থাৎ, **HyperText** হলো এমন লেখা, যা Link-এর মাধ্যমে অন্য তথ্য বা ডকুমেন্টের সাথে সংযুক্ত থাকে।

## HyperText-এর বৈশিষ্ট্য

1. এতে Hyperlink থাকে।
2. একটি Page থেকে অন্য Page-এ যাওয়া যায়।
3. একই Page-এর নির্দিষ্ট অংশে Jump করা যায়।
4. Website-গুলোর মধ্যে সংযোগ (Navigation) তৈরি করে।
5. ব্যবহারকারীর জন্য তথ্য খুঁজে পাওয়া সহজ করে।
6. World Wide Web (WWW)-এর মূল ভিত্তিগুলোর একটি।

## HyperText কীভাবে কাজ করে?

যখন একটি Web Page-এ কোনো Link যোগ করা হয়, তখন Browser সেই Link-টিকে শনাক্ত করে।

ব্যবহারকারী যদি Link-এ Click করে, তাহলে Browser—

* অন্য একটি Web Page খুলতে পারে।
* একই Page-এর নির্দিষ্ট অংশে যেতে পারে।
* অন্য কোনো Website-এ নিয়ে যেতে পারে।
* কোনো File Download করতে পারে।
* Email পাঠানোর জন্য Email Client খুলতে পারে।

এই পুরো প্রক্রিয়াটি HyperText-এর মাধ্যমে সম্পন্ন হয়।

## বাস্তব উদাহরণ

ধরুন আপনি একটি বই পড়ছেন।

সাধারণ বইয়ে যদি লেখা থাকে—

"বিস্তারিত জানতে ২৫ নম্বর পৃষ্ঠা দেখুন।"

তাহলে আপনাকে নিজেই ২৫ নম্বর পৃষ্ঠা খুঁজে বের করতে হবে।

কিন্তু একটি Website-এ যদি একই লেখার সাথে একটি Link থাকে, তাহলে সেই Link-এ Click করলেই Browser সরাসরি নির্দিষ্ট Page বা তথ্য খুলে দেয়।

এটাই HyperText-এর সবচেয়ে বড় সুবিধা।

## HTML-এ HyperText-এর উদাহরণ

```html
<a href="about.html">About Us</a>
```

এখানে—

* `<a>` হলো Anchor Tag।
* `href` Attribute-এর মাধ্যমে Link-এর ঠিকানা দেওয়া হয়।
* `About Us` হলো ব্যবহারকারী যে লেখাটি দেখতে পায়।
* Link-এ Click করলে Browser `about.html` Page-এ নিয়ে যাবে।

## HyperText-এর ব্যবহার

* Website-এর বিভিন্ন Page সংযুক্ত করা।
* অন্য Website-এ যাওয়া।
* একই Page-এর নির্দিষ্ট অংশে Jump করা।
* PDF, Image বা অন্য File Download করা।
* Email Link তৈরি করা।
* Navigation Menu তৈরি করা।

## HyperText-এর সুবিধা

* দ্রুত তথ্যে পৌঁছানো যায়।
* Website ব্যবহার করা সহজ হয়।
* Page-এর মধ্যে সহজে Navigation করা যায়।
* তথ্য একে অপরের সাথে সংযুক্ত রাখা যায়।
* ব্যবহারকারীর সময় ও পরিশ্রম কমে।

## সারসংক্ষেপ

HyperText হলো এমন একটি Text, যার মধ্যে Link থাকে। এই Link-এর মাধ্যমে ব্যবহারকারী একটি Page থেকে অন্য Page, একটি Website থেকে অন্য Website অথবা একই Page-এর নির্দিষ্ট অংশে সহজেই যেতে পারে। HTML-এর **HyperText** অংশটি এই Link-ভিত্তিক তথ্য সংযোগের ধারণাকেই বোঝায় এবং এটি World Wide Web-এর অন্যতম মৌলিক ভিত্তি।






# > 1.3 Markup Language কী?

Markup Language হলো এমন একটি **Language**, যা **Tag (ট্যাগ)** ব্যবহার করে কোনো Document-এর **Structure (কাঠামো)**, **Meaning (অর্থ)** এবং **Content-এর বিন্যাস** নির্ধারণ করে। এটি কনটেন্ট কীভাবে সাজানো থাকবে এবং কোন অংশের কী ভূমিকা, তা Browser বা অন্য সফটওয়্যারকে বুঝতে সাহায্য করে।

Markup Language নিজে কোনো কাজ (Logic) সম্পাদন করে না, যেমন গণনা করা, সিদ্ধান্ত নেওয়া বা Loop চালানো। বরং এটি শুধু কনটেন্টকে **চিহ্নিত (Mark Up)** করে এবং তার গঠন নির্ধারণ করে।

HTML একটি **Markup Language**, কারণ এটি বিভিন্ন Tag ব্যবহার করে Web Page-এর কাঠামো তৈরি করে।

## Markup শব্দের অর্থ

**Markup** শব্দটি এসেছে **"Mark Up"** থেকে, যার অর্থ হলো **চিহ্নিত করা (Mark)** বা **ট্যাগের মাধ্যমে কোনো কনটেন্টকে নির্দিষ্ট পরিচয় দেওয়া**।

উদাহরণস্বরূপ—

* এটি একটি Heading।
* এটি একটি Paragraph।
* এটি একটি Image।
* এটি একটি Link।
* এটি একটি Table।

অর্থাৎ, Markup-এর কাজ হলো প্রতিটি Content-এর ধরন (Type) নির্ধারণ করা।

## Language শব্দের অর্থ

Language বলতে এমন একটি নিয়মভিত্তিক (Rule-based) পদ্ধতিকে বোঝায়, যার মাধ্যমে নির্দিষ্ট Syntax অনুসরণ করে তথ্য লেখা হয় এবং Browser বা Software তা বুঝতে পারে।

## Markup Language কীভাবে কাজ করে?

Markup Language-এ বিভিন্ন **Tag** ব্যবহার করে কনটেন্টকে ঘিরে (Wrap) দেওয়া হয়।

Browser যখন HTML File পড়ে, তখন সে প্রতিটি Tag-এর অর্থ বুঝে এবং সেই অনুযায়ী Web Page প্রদর্শন করে।

উদাহরণস্বরূপ—

* `<h1>` → বড় শিরোনাম
* `<p>` → অনুচ্ছেদ
* `<img>` → ছবি
* `<a>` → লিংক

Browser Tag-গুলো ব্যবহারকারীর সামনে দেখায় না; বরং Tag-এর নির্দেশনা অনুযায়ী কনটেন্ট প্রদর্শন করে।

## HTML-এ Markup-এর উদাহরণ

```html
<h1>Welcome to My Website</h1>

<p>This is my first HTML page.</p>
```

উপরের কোডে—

* `<h1>` Tag দ্বারা "Welcome to My Website" একটি প্রধান শিরোনাম হিসেবে চিহ্নিত হয়েছে।
* `<p>` Tag দ্বারা "This is my first HTML page." একটি Paragraph হিসেবে চিহ্নিত হয়েছে।

এখানে Tag-গুলো কনটেন্টকে **Mark Up** করেছে।

## Markup Language-এর বৈশিষ্ট্য

1. Tag ব্যবহার করে Content-এর ধরন নির্ধারণ করে।
2. Document-এর Structure তৈরি করে।
3. Browser-কে Content-এর অর্থ বুঝতে সাহায্য করে।
4. এটি Programming Language নয়।
5. Logic, Calculation বা Decision নেওয়ার কাজ করে না।
6. মানুষ এবং Browser—উভয়ের জন্য Document বোঝা সহজ করে।

## Markup Language-এর ব্যবহার

* Web Page-এর Structure তৈরি করা।
* Heading, Paragraph, Image, Table, Form ইত্যাদি নির্ধারণ করা।
* Content-কে সঠিকভাবে সংগঠিত করা।
* Browser-কে Document কীভাবে প্রদর্শন করতে হবে তা জানানো।
* Search Engine ও অন্যান্য Software-কে Content বুঝতে সাহায্য করা।

## Markup Language বনাম Programming Language

| Markup Language          | Programming Language                            |
| ------------------------ | ----------------------------------------------- |
| Structure তৈরি করে       | Logic ও Functionality তৈরি করে                  |
| Tag ব্যবহার করে          | Variables, Functions, Loops ইত্যাদি ব্যবহার করে |
| Calculation করতে পারে না | Calculation করতে পারে                           |
| Decision নিতে পারে না    | Decision নিতে পারে                              |
| HTML, XML উদাহরণ         | JavaScript, Python, Java, C++ উদাহরণ            |

## বাস্তব উদাহরণ

ধরুন, আপনি একটি বই প্রকাশ করছেন।

* **Markup Language** হলো সেই নির্দেশনা, যেখানে লেখা থাকে—"এটি শিরোনাম হবে", "এটি অনুচ্ছেদ হবে", "এখানে ছবি থাকবে"।
* **Programming Language** হলো সেই ব্যবস্থা, যা বইয়ের ডিজিটাল অ্যাপে Search, Bookmark, Login বা অন্যান্য কাজ পরিচালনা করে।

অর্থাৎ—

* **Markup Language** → কী আছে এবং কীভাবে সাজানো হবে।
* **Programming Language** → কী কাজ করবে এবং কীভাবে কাজ করবে।

## সারসংক্ষেপ

Markup Language হলো এমন একটি ভাষা, যা Tag ব্যবহার করে কোনো Document-এর Structure ও অর্থ নির্ধারণ করে। HTML একটি Markup Language, কারণ এটি বিভিন্ন Tag-এর মাধ্যমে Web Page-এর প্রতিটি অংশকে চিহ্নিত করে এবং Browser-কে জানায় কীভাবে সেই Page প্রদর্শন করতে হবে। এটি Programming Language নয়, বরং Web Development-এর ভিত্তি হিসেবে Web Page-এর কাঠামো তৈরি করার জন্য ব্যবহৃত হয়।






# > 1.4 HTML এর কাজ কী?

HTML-এর প্রধান কাজ হলো **Web Page-এর Structure (কাঠামো) তৈরি করা**। এটি বিভিন্ন **Tag (ট্যাগ)** ব্যবহার করে Browser-কে জানায় কোন অংশটি Heading, কোনটি Paragraph, কোনটি Image, কোনটি Link, কোনটি Table বা Form।

HTML কোনো ওয়েবসাইটের ডিজাইন (Design) বা কার্যকারিতা (Functionality) তৈরি করে না। এটি শুধুমাত্র ওয়েব পেজের **Content** এবং **Structure** নির্ধারণ করে।

সহজ ভাষায়, HTML হলো একটি ওয়েবসাইটের **ভিত্তি (Foundation)** বা **কঙ্কাল (Skeleton)**। HTML ছাড়া কোনো Web Page-এর গঠন তৈরি করা সম্ভব নয়।

## HTML-এর প্রধান কাজ

### 1. Web Page-এর Structure তৈরি করা

HTML-এর সবচেয়ে গুরুত্বপূর্ণ কাজ হলো একটি Web Page-এর কাঠামো তৈরি করা।

উদাহরণস্বরূপ—

* Header
* Navigation Menu
* Main Content
* Sidebar
* Footer

এসব অংশ HTML-এর মাধ্যমেই তৈরি করা হয়।

### 2. Content প্রদর্শন করা

HTML ব্যবহার করে বিভিন্ন ধরনের তথ্য Browser-এ প্রদর্শন করা যায়।

যেমন—

* Heading
* Paragraph
* Text
* Quote
* Code
* List

### 3. Link তৈরি করা

HTML-এর মাধ্যমে একটি Page থেকে অন্য Page অথবা একটি Website থেকে অন্য Website-এ যাওয়ার জন্য Hyperlink তৈরি করা যায়।

উদাহরণ—

* Home Page
* About Page
* Contact Page
* External Website

### 4. Image যোগ করা

HTML-এর মাধ্যমে Web Page-এ ছবি যুক্ত করা যায়।

ছবির জন্য `<img>` Tag ব্যবহার করা হয়।

### 5. Audio ও Video যুক্ত করা

HTML ব্যবহার করে Browser-এর মধ্যেই Audio এবং Video চালানো যায়।

এর জন্য—

* `<audio>`
* `<video>`

Tag ব্যবহার করা হয়।

### 6. Table তৈরি করা

তথ্যকে সারি (Row) এবং কলাম (Column)-এ সাজানোর জন্য HTML Table ব্যবহার করা হয়।

যেমন—

* Student List
* Product List
* Employee Data
* Result Sheet

### 7. List তৈরি করা

HTML দিয়ে বিভিন্ন ধরনের তালিকা তৈরি করা যায়।

যেমন—

* Ordered List (Number)
* Unordered List (Bullet)
* Description List

### 8. Form তৈরি করা

ব্যবহারকারীর কাছ থেকে তথ্য নেওয়ার জন্য HTML Form ব্যবহার করা হয়।

যেমন—

* Login Form
* Registration Form
* Contact Form
* Search Box

### 9. Browser-কে Content-এর অর্থ বোঝানো

HTML Tag Browser-কে জানায় কোন Content কী ধরনের।

উদাহরণ—

* `<h1>` → প্রধান শিরোনাম
* `<p>` → Paragraph
* `<img>` → Image
* `<button>` → Button

Browser সেই অনুযায়ী Page প্রদর্শন করে।

### 10. Semantic Structure তৈরি করা

HTML5-এ Semantic Tag ব্যবহার করে Page-এর বিভিন্ন অংশকে অর্থপূর্ণভাবে ভাগ করা যায়।

যেমন—

* `<header>`
* `<nav>`
* `<main>`
* `<section>`
* `<article>`
* `<aside>`
* `<footer>`

এতে Browser, Search Engine এবং Developer—সবার জন্য Page বোঝা সহজ হয়।

## HTML কী কী তৈরি করতে পারে?

HTML ব্যবহার করে—

* সাধারণ Web Page
* Portfolio Website
* Blog
* News Website
* E-commerce Website-এর Structure
* School বা College Website
* Company Website
* Landing Page
* Documentation Page
* Web Application-এর Structure

তৈরি করা যায়।

## HTML কী করতে পারে না?

HTML একা নিচের কাজগুলো করতে পারে না—

* Web Page-কে সুন্দরভাবে ডিজাইন করতে পারে না।
* Animation তৈরি করতে পারে না।
* Button-এ Click করলে Logic চালাতে পারে না।
* Database-এর সাথে যোগাযোগ করতে পারে না।
* User Authentication করতে পারে না।
* Dynamic Data তৈরি করতে পারে না।

এসব কাজের জন্য CSS, JavaScript এবং Backend Technology ব্যবহার করা হয়।

## HTML, CSS এবং JavaScript-এর কাজ

| প্রযুক্তি  | প্রধান কাজ                                       |
| ---------- | ------------------------------------------------ |
| HTML       | Web Page-এর Structure তৈরি করে                   |
| CSS        | Web Page-এর Design ও Styling করে                 |
| JavaScript | Web Page-এ Functionality ও Interactivity যোগ করে |

এই তিনটি প্রযুক্তি একসাথে ব্যবহার করে একটি সম্পূর্ণ আধুনিক Website তৈরি করা হয়।

## বাস্তব উদাহরণ

ধরুন, আপনি একটি বাড়ি তৈরি করছেন।

* **HTML** → বাড়ির ভিত্তি, দেয়াল, দরজা, জানালা এবং কক্ষের কাঠামো তৈরি করে।
* **CSS** → বাড়ির রং, টাইলস, সাজসজ্জা এবং সৌন্দর্য বৃদ্ধি করে।
* **JavaScript** → দরজার অটোমেশন, লাইট নিয়ন্ত্রণ, স্মার্ট ডিভাইস এবং অন্যান্য কার্যকরী ব্যবস্থা পরিচালনা করে।

অর্থাৎ—

* **HTML** → কী থাকবে এবং কোথায় থাকবে।
* **CSS** → দেখতে কেমন হবে।
* **JavaScript** → কীভাবে কাজ করবে।

## সারসংক্ষেপ

HTML-এর মূল কাজ হলো একটি Web Page-এর **Structure** তৈরি করা এবং Browser-কে জানানো কোন অংশটি কী ধরনের Content। HTML ব্যবহার করে Heading, Paragraph, Image, Link, Table, List, Form, Audio, Video এবং অন্যান্য উপাদান তৈরি করা যায়। তবে HTML শুধুমাত্র কাঠামো তৈরি করে; Design-এর জন্য CSS এবং Functionality-এর জন্য JavaScript ব্যবহার করা হয়।






# > 1.5 HTML Versions

HTML Versions বলতে HTML-এর বিভিন্ন **সংস্করণ (Version)** বোঝায়। Web Technology-এর উন্নয়নের সাথে সাথে HTML-এ নতুন নতুন Feature, Tag এবং API যুক্ত হয়েছে। তাই সময়ের প্রয়োজন অনুযায়ী HTML-এর একাধিক Version প্রকাশ করা হয়েছে।

প্রতিটি নতুন Version আগের Version-এর সীমাবদ্ধতা দূর করে আরও উন্নত সুবিধা প্রদান করেছে। বর্তমানে **HTML5** হলো সবচেয়ে আধুনিক এবং সর্বাধিক ব্যবহৃত Version।

## HTML Versions-এর ইতিহাস

| Version   | প্রকাশের বছর                   | প্রধান বৈশিষ্ট্য                                                                                                                                                    |
| --------- | ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| HTML 1.0  | 1993                           | HTML-এর প্রথম সংস্করণ। শুধুমাত্র মৌলিক Tag ব্যবহার করে সাধারণ Web Page তৈরি করা যেত।                                                                                |
| HTML 2.0  | 1995                           | HTML-এর প্রথম Official Standard। Form, Table-এর প্রাথমিক সমর্থন এবং আরও স্থিতিশীল Specification যুক্ত হয়।                                                          |
| HTML 3.2  | 1997                           | Table, Applet, Font Tag এবং আরও উন্নত Presentation Feature যোগ হয়।                                                                                                 |
| HTML 4.01 | 1999                           | CSS সমর্থন উন্নত করা হয়, Script ব্যবহার সহজ হয় এবং Semantic Structure-এর দিকে গুরুত্ব দেওয়া শুরু হয়।                                                            |
| XHTML 1.0 | 2000                           | HTML-কে XML-এর নিয়ম অনুযায়ী আরও Strict Syntax-এ লেখা শুরু হয়। প্রতিটি Tag সঠিকভাবে বন্ধ (Close) করতে হতো এবং ছোট হাতের (Lowercase) অক্ষর ব্যবহার করার নিয়ম ছিল। |
| HTML5     | 2014 (Official Recommendation) | Semantic Tag, Audio, Video, Canvas, SVG, Geolocation, Local Storage, Drag & Drop এবং অসংখ্য আধুনিক Web Feature যুক্ত হয়।                                           |

## প্রতিটি Version-এর সংক্ষিপ্ত পরিচয়

### HTML 1.0 (1993)

HTML-এর প্রথম Version।

এতে শুধুমাত্র মৌলিক উপাদান ছিল, যেমন—

* Heading
* Paragraph
* Link
* Basic Text Formatting

এটি খুবই সীমিত ক্ষমতাসম্পন্ন ছিল।

### HTML 2.0 (1995)

এটি HTML-এর প্রথম Official Standard।

নতুন কিছু সুবিধা যুক্ত হয়—

* Form
* উন্নত Link
* Input Field
* আরও নির্ভরযোগ্য Browser Support

### HTML 3.2 (1997)

এই Version-এ Web Page আরও আকর্ষণীয় করা সম্ভব হয়।

নতুন Feature—

* Table
* Font Tag
* Applet
* উন্নত Layout Support

### HTML 4.01 (1999)

দীর্ঘ সময় ধরে এটি সবচেয়ে জনপ্রিয় Version ছিল।

নতুন সুবিধা—

* CSS-এর ভালো সমর্থন
* JavaScript Integration
* Accessibility উন্নয়ন
* Internationalization Support

Presentation-এর পরিবর্তে Structure-এর উপর বেশি গুরুত্ব দেওয়া শুরু হয়।

### XHTML 1.0 (2000)

XHTML-এর পূর্ণরূপ হলো **Extensible HyperText Markup Language**।

এটি HTML এবং XML-এর সমন্বয়ে তৈরি।

XHTML-এ কিছু কঠোর নিয়ম ছিল—

* সব Tag অবশ্যই Close করতে হবে।
* সব Tag ছোট হাতের অক্ষরে লিখতে হবে।
* Attribute-এর মান সবসময় Quote-এর মধ্যে লিখতে হবে।
* Document অবশ্যই Well-Formed হতে হবে।

উদাহরণ—

```html id="j2m8c4"
<br />
<img src="photo.jpg" alt="Photo" />
```

### HTML5 (2014 - বর্তমান)

HTML5 হলো বর্তমান সময়ের Standard Version এবং আধুনিক Web Development-এর ভিত্তি।

HTML5-এ যুক্ত হয়েছে—

* Semantic Tag
* Audio Support
* Video Support
* Canvas Graphics
* SVG Graphics
* Local Storage
* Session Storage
* Geolocation API
* Drag and Drop API
* Web Storage
* Offline Support
* Responsive Web-এর জন্য উন্নত সুবিধা

HTML5-এর কিছু গুরুত্বপূর্ণ Semantic Tag—

* `<header>`
* `<nav>`
* `<main>`
* `<section>`
* `<article>`
* `<aside>`
* `<footer>`

## HTML5 কেন সবচেয়ে গুরুত্বপূর্ণ?

HTML5-এর মাধ্যমে—

* Plugin ছাড়াই Audio ও Video চালানো যায়।
* আরও পরিষ্কার ও অর্থবহ Structure তৈরি করা যায়।
* Mobile Device ভালোভাবে Support করে।
* আধুনিক Browser-এর সাথে সম্পূর্ণ সামঞ্জস্যপূর্ণ।
* Web Application তৈরি করা সহজ হয়েছে।
* Performance এবং User Experience উন্নত হয়েছে।

বর্তমানে নতুন Website বা Web Application তৈরির জন্য HTML5 ব্যবহার করাই Standard Practice।

বর্তমানে **HTML5** ব্যবহার করা উচিত, কারণ—

* Responsive Web Design-এর জন্য উপযোগী।
* CSS3 এবং JavaScript-এর সাথে সবচেয়ে ভালোভাবে কাজ করে।
* ভবিষ্যতের Web Development-এর জন্য এটি Standard Version।

## সারসংক্ষেপ

HTML-এর শুরু হয় **HTML 1.0** দিয়ে এবং ধাপে ধাপে উন্নত হতে হতে বর্তমান **HTML5**-এ পৌঁছেছে। প্রতিটি নতুন Version আগের তুলনায় আরও বেশি Feature, উন্নত Structure এবং ভালো Browser Support প্রদান করেছে। বর্তমানে Web Development-এর জন্য **HTML5**-ই সবচেয়ে আধুনিক, শক্তিশালী এবং সর্বাধিক ব্যবহৃত HTML Version।






# > 1.6 HTML File Extension

HTML File Extension হলো একটি HTML ফাইলের **শেষ অংশ (Suffix)**, যা অপারেটিং সিস্টেম এবং Web Browser-কে জানায় যে এটি একটি HTML Document। File Extension দেখে Browser বুঝতে পারে কীভাবে ফাইলটি খুলতে এবং প্রদর্শন করতে হবে।

HTML ফাইলের সবচেয়ে প্রচলিত Extension হলো—

* `.html`
* `.htm`

বর্তমানে **`.html`** ব্যবহার করাই Standard এবং সবচেয়ে বেশি প্রচলিত।

## File Extension কী?

File Extension হলো কোনো ফাইলের নামের শেষে থাকা অংশ, যা একটি ডট (`.`)-এর পরে লেখা হয়।

উদাহরণ—

```text id="3h7f2a"
index.html
about.html
contact.html
style.css
script.js
image.png
```

এখানে—

* `.html` → HTML File
* `.css` → CSS File
* `.js` → JavaScript File
* `.png` → Image File

অর্থাৎ, Extension দেখে বোঝা যায় ফাইলটির ধরন (Type) কী।

## HTML File Extension

HTML-এর জন্য দুটি Extension ব্যবহৃত হয়—

### 1. `.html`

এটি HTML-এর Official এবং সর্বাধিক ব্যবহৃত Extension।

উদাহরণ—

```text id="8k5m1d"
index.html
home.html
about.html
contact.html
login.html
```

বর্তমানে নতুন সব Website এবং Web Project-এ `.html` ব্যবহার করা হয়।

### 2. `.htm`

`.htm` হলো HTML-এর পুরোনো Extension।

এটি মূলত পুরোনো **MS-DOS** এবং **Windows**-এর সেই সময় ব্যবহৃত হতো, যখন File Extension সর্বোচ্চ ৩ অক্ষরের হতে পারত। তাই `.html`-এর পরিবর্তে `.htm` ব্যবহার করা হতো।

উদাহরণ—

```text id="m9q4r8"
index.htm
home.htm
about.htm
```

বর্তমান Browser-গুলো `.htm` এবং `.html`—উভয়ই সমর্থন করে, তবে `.html` ব্যবহার করাই উত্তম।

## `.html` এবং `.htm`-এর মধ্যে পার্থক্য

| `.html`                          | `.htm`                               |
| -------------------------------- | ------------------------------------ |
| বর্তমান Standard Extension       | পুরোনো Extension                     |
| ৫ অক্ষরের Extension              | ৩ অক্ষরের Extension                  |
| বর্তমানে সবচেয়ে বেশি ব্যবহৃত    | বর্তমানে খুব কম ব্যবহৃত              |
| নতুন Project-এর জন্য Recommended | শুধুমাত্র পুরোনো Project-এ দেখা যায় |

কার্যকারিতার দিক থেকে Browser-এর কাছে দুটিই একই; পার্থক্য শুধুমাত্র নামের।

## HTML File কীভাবে তৈরি করবেন?

ধাপ ১: একটি Text Editor খুলুন।

যেমন—

* Visual Studio Code (VS Code)
* Notepad++
* Sublime Text
* Notepad

ধাপ ২: HTML Code লিখুন।

```html id="mx2v7p"
<!DOCTYPE html>
<html>
<head>
    <title>My First Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
</body>
</html>
```

ধাপ ৩: ফাইলটি `.html` Extension দিয়ে সংরক্ষণ করুন।

উদাহরণ—

```text id="0t6w9n"
index.html
```

ধাপ ৪: ফাইলটিতে Double Click করুন অথবা Browser-এ Open করুন।

Browser HTML Code পড়ে Web Page হিসেবে প্রদর্শন করবে।

## HTML File-এর সাধারণ Naming Convention

HTML ফাইলের নাম রাখার সময় কিছু ভালো অভ্যাস অনুসরণ করা উচিত—

* ছোট হাতের অক্ষর (lowercase) ব্যবহার করুন।
* ফাইলের নামে স্পেস ব্যবহার না করাই ভালো।
* একাধিক শব্দ হলে Hyphen (`-`) ব্যবহার করুন।
* নাম অর্থবহ (Meaningful) রাখুন।

ভালো উদাহরণ—

```text id="k8z1s4"
index.html
about.html
contact.html
login.html
user-profile.html
product-list.html
```

এড়িয়ে চলুন—

```text id="e5p2x7"
My Page.HTML
About Us.html
New File (1).html
```

## Website-এ `index.html` কেন গুরুত্বপূর্ণ?

বেশিরভাগ Web Server-এ `index.html` হলো **Default Home Page**।

যখন ব্যবহারকারী শুধুমাত্র একটি Website-এর Domain Address লিখে প্রবেশ করে, তখন Server স্বয়ংক্রিয়ভাবে `index.html` ফাইলটি খুঁজে Browser-এ প্রদর্শন করে।

উদাহরণ—

```text id="n4y8b6"
https://example.com/
```

যদি Root Folder-এ `index.html` থাকে, তাহলে সেটিই Home Page হিসেবে খুলবে।

## HTML File Extension-এর সুবিধা

* Browser সহজেই HTML File চিনতে পারে।
* Web Page হিসেবে সঠিকভাবে Render হয়।
* সকল আধুনিক Browser সমর্থন করে।
* Web Server সহজে Default Page শনাক্ত করতে পারে।
* Web Development-এর Standard অনুসরণ করা যায়।

## সারসংক্ষেপ

HTML File Extension হলো HTML Document-এর পরিচয় বহনকারী অংশ। HTML-এর জন্য `.html` এবং `.htm`—দুটি Extension থাকলেও বর্তমানে **`.html`**-ই Standard এবং সবচেয়ে বেশি ব্যবহৃত। নতুন Web Project তৈরি করার সময় সবসময় `.html` Extension ব্যবহার করা উচিত, কারণ এটি আধুনিক Browser, Web Server এবং Web Development-এর বর্তমান মানদণ্ডের সাথে সম্পূর্ণ সামঞ্জস্যপূর্ণ।






# > 1.7 HTML Editor

HTML Editor হলো এমন একটি **Software** বা **Application**, যার মাধ্যমে HTML Code লেখা, সম্পাদনা (Edit), সংরক্ষণ (Save) এবং পরিচালনা (Manage) করা যায়।

সহজ ভাষায়, HTML Editor হলো এমন একটি টুল যেখানে আপনি HTML, CSS, JavaScript এবং অন্যান্য Web Development-এর Code লিখে Website তৈরি করতে পারেন।

যদিও HTML Code সাধারণ **Text Editor**-এও লেখা যায়, তবে একটি **Code Editor** ব্যবহার করলে কাজ অনেক সহজ, দ্রুত এবং কম ভুল হয়।

## HTML Editor কেন ব্যবহার করা হয়?

HTML Editor ব্যবহার করার প্রধান উদ্দেশ্য হলো সহজে এবং দক্ষতার সাথে Code লেখা ও পরিচালনা করা।

এটি ব্যবহার করে আপনি—

* HTML Code লিখতে পারবেন।
* Code সম্পাদনা (Edit) করতে পারবেন।
* File Save করতে পারবেন।
* Project পরিচালনা করতে পারবেন।
* Code-এর ভুল (Error) সহজে খুঁজে বের করতে পারবেন।
* দ্রুত এবং সুন্দরভাবে Coding করতে পারবেন।

## HTML Editor-এর প্রধান বৈশিষ্ট্য

একটি ভালো HTML Editor-এ সাধারণত নিচের সুবিধাগুলো থাকে—

### 1. Syntax Highlighting

Editor বিভিন্ন ধরনের Code-কে ভিন্ন ভিন্ন রঙে দেখায়।

যেমন—

* HTML Tag
* Attribute
* String
* Comment

এতে Code পড়া ও বোঝা সহজ হয়।

### 2. Auto Completion (IntelliSense)

Code লেখার সময় Editor স্বয়ংক্রিয়ভাবে Tag, Attribute বা Code Suggest করে।

উদাহরণ—

আপনি যদি লিখেন—

```text id="6v8m2p"
<di
```

তাহলে Editor Suggest করবে—

```text id="9q1t4k"
<div>
```

এতে Coding-এর গতি বাড়ে এবং ভুল কম হয়।

### 3. Auto Closing Tag

Opening Tag লিখলে অনেক Editor স্বয়ংক্রিয়ভাবে Closing Tag তৈরি করে।

উদাহরণ—

```html id="h7n3x5"
<p></p>
```

### 4. Error Detection

Editor অনেক সাধারণ ভুল (Syntax Error) আগে থেকেই শনাক্ত করে দেখিয়ে দেয়।

ফলে ভুল দ্রুত ঠিক করা যায়।

### 5. File Explorer

Project-এর সব File এবং Folder এক জায়গা থেকে দেখা ও পরিচালনা করা যায়।

### 6. Search and Replace

একটি শব্দ বা Code একসাথে পুরো Project-এ খুঁজে বের করা এবং প্রয়োজনে পরিবর্তন করা যায়।

### 7. Extensions

অনেক Editor-এ অতিরিক্ত Feature যোগ করার জন্য Extension বা Plugin ব্যবহার করা যায়।

যেমন—

* Live Server
* Prettier
* Auto Rename Tag
* Git Integration

# > 1.8 HTML Browser

HTML Browser হলো এমন একটি **Software Application**, যা HTML File পড়ে (Read), বুঝে (Parse) এবং সেটিকে একটি সুন্দর **Web Page** হিসেবে ব্যবহারকারীর সামনে প্রদর্শন (Render) করে।

সহজ ভাষায়, HTML Browser হলো এমন একটি সফটওয়্যার যা HTML Code-কে মানুষের পড়ার উপযোগী Web Page-এ রূপান্তর করে।

Browser HTML Code সরাসরি দেখায় না। বরং HTML-এর Tag, CSS-এর Style এবং JavaScript-এর Code বিশ্লেষণ করে (Interpret) একটি সম্পূর্ণ Web Page তৈরি করে।

## Browser কী?

Browser (Web Browser) হলো একটি সফটওয়্যার, যার মাধ্যমে আমরা Website এবং Web Application ব্যবহার করতে পারি।

Browser-এর প্রধান কাজ হলো—

* HTML File পড়া।
* CSS File প্রয়োগ করা।
* JavaScript চালানো।
* Web Page প্রদর্শন করা।
* ব্যবহারকারীর Input গ্রহণ করা।
* Web Server-এর সাথে যোগাযোগ করা।

## HTML Browser কীভাবে কাজ করে?

যখন আপনি একটি HTML File Open করেন অথবা কোনো Website-এ প্রবেশ করেন, তখন Browser নিচের ধাপগুলো অনুসরণ করে।

### ধাপ ১: HTML File গ্রহণ করা

Browser প্রথমে HTML File পড়ে।

এটি হতে পারে—

* Local Computer থেকে
* Web Server থেকে
* Internet-এর মাধ্যমে

### ধাপ ২: Parsing (HTML বিশ্লেষণ)

Browser HTML Code-এর প্রতিটি Tag পড়ে এবং বুঝে।

যেমন—

```html id="b7k2m8"
<h1>Welcome</h1>
<p>Hello World</p>
```

Browser বুঝে—

* `<h1>` একটি Heading।
* `<p>` একটি Paragraph।

### ধাপ ৩: DOM (Document Object Model) তৈরি করা

HTML Parse করার পরে Browser একটি **DOM Tree** তৈরি করে।

DOM হলো HTML Document-এর একটি Object আকারের কাঠামো, যেখানে প্রতিটি HTML Element একটি Node হিসেবে থাকে।

উদাহরণ—

```text id="t9q4d1"
html
│
├── head
│
└── body
    │
    ├── h1
    │
    └── p
```

JavaScript এই DOM ব্যবহার করে Web Page পরিবর্তন করতে পারে।

### ধাপ ৪: CSS প্রয়োগ করা

যদি HTML-এর সাথে CSS যুক্ত থাকে, তাহলে Browser CSS File পড়ে এবং প্রতিটি Element-এর Style নির্ধারণ করে।

যেমন—

* রং (Color)
* আকার (Size)
* Font
* Margin
* Padding

### ধাপ ৫: Layout তৈরি করা

Browser নির্ধারণ করে—

* কোন Element কোথায় থাকবে।
* কত বড় হবে।
* কত দূরত্ব থাকবে।
* কীভাবে Screen-এ সাজানো হবে।

এটিকে **Layout** বা **Reflow** বলা হয়।

### ধাপ ৬: Rendering (আঁকা)

সবশেষে Browser Screen-এ Web Page আঁকে (Render করে)।

এখন ব্যবহারকারী সম্পূর্ণ Web Page দেখতে পারে।

## Browser Rendering Process

HTML থেকে Web Page প্রদর্শনের পুরো প্রক্রিয়াটি সংক্ষেপে—


কিন্তু সেই Code-এর ফলাফল (Web Page) দেখতে হলে অবশ্যই একটি Browser প্রয়োজন।

# > 1.9 HTML কীভাবে কাজ করে?

HTML কীভাবে কাজ করে তা বোঝার জন্য প্রথমে জানতে হবে যে **HTML নিজে কোনো Program চালায় না**। HTML-এর কাজ হলো একটি Web Page-এর **Structure (কাঠামো)** তৈরি করা এবং Browser-কে জানানো কোন অংশটি কী ধরনের Content।

যখন একজন ব্যবহারকারী কোনো Website খুলে, তখন Browser এবং Web Server একসাথে কাজ করে HTML File সংগ্রহ করে, সেটিকে বিশ্লেষণ (Parse) করে এবং শেষে একটি সুন্দর Web Page হিসেবে প্রদর্শন (Render) করে।

সহজভাবে বলতে গেলে—

> **HTML File → Browser পড়ে → HTML বিশ্লেষণ করে → CSS প্রয়োগ করে → JavaScript চালায় → Web Page প্রদর্শন করে।**

## HTML কাজ করার সম্পূর্ণ ধাপ

### ধাপ ১: ব্যবহারকারী Website Request করে

প্রথমে ব্যবহারকারী Browser-এর Address Bar-এ একটি Website-এর URL লিখে অথবা কোনো Link-এ Click করে।

উদাহরণ—

```text id="a1k9p3"
https://example.com
```

Browser তখন Website-এর জন্য Request পাঠায়।

### ধাপ ২: Browser Server-এ Request পাঠায়

Browser **HTTP** বা **HTTPS** Protocol ব্যবহার করে Web Server-এর কাছে Request পাঠায়।

উদাহরণ—

```text id="d8m4q2"
GET / HTTP/1.1
```

এটিকে **HTTP Request** বলা হয়।

### ধাপ ৩: Web Server HTML File পাঠায়

Server Request পাওয়ার পরে প্রয়োজনীয় HTML File খুঁজে Browser-এর কাছে পাঠায়।

উদাহরণ—

```text id="g7x5n8"
index.html
```

এই File-এর সাথে CSS, JavaScript, Image এবং অন্যান্য Resource-এর তথ্যও থাকতে পারে।

### ধাপ ৪: Browser HTML Parse করে

Browser HTML File-এর প্রতিটি Tag একে একে পড়ে এবং তার অর্থ বুঝতে চেষ্টা করে।

উদাহরণ—

```html id="u2r8c6"
<h1>Welcome</h1>

<p>Hello World</p>
```

Browser বুঝে—

* `<h1>` → এটি একটি প্রধান শিরোনাম।
* `<p>` → এটি একটি Paragraph।

এই বিশ্লেষণ করার প্রক্রিয়াকে **Parsing** বলা হয়।

### ধাপ ৫: Browser DOM Tree তৈরি করে

Parsing শেষ হলে Browser HTML Document-এর একটি **DOM (Document Object Model)** তৈরি করে।

DOM হলো HTML-এর একটি Tree Structure, যেখানে প্রতিটি HTML Element একটি Node হিসেবে থাকে।

উদাহরণ—

```text id="f4v7b1"
html
│
├── head
│
│   └── title
│
└── body
    │
    ├── h1
    │
    └── p
```

JavaScript এই DOM ব্যবহার করে Web Page পরিবর্তন করতে পারে।

### ধাপ ৬: Browser CSS File পড়ে

যদি HTML-এর সাথে CSS যুক্ত থাকে, তাহলে Browser CSS File Download করে।

উদাহরণ—

```html id="e6w3h9"
<link rel="stylesheet" href="style.css">
```

এরপর CSS Parse করে প্রতিটি Element-এর Style নির্ধারণ করে।

যেমন—

* Color
* Font
* Margin
* Padding
* Width
* Height

### ধাপ ৭: Browser Render Tree তৈরি করে

Browser HTML-এর DOM এবং CSS-এর Style একত্র করে একটি **Render Tree** তৈরি করে।

Render Tree-তে শুধুমাত্র Screen-এ প্রদর্শিত হওয়া Element-গুলো থাকে।

### ধাপ ৮: Layout (Reflow) গণনা করা

Browser নির্ধারণ করে—

* কোন Element কোথায় থাকবে।
* কত বড় হবে।
* কত দূরে থাকবে।
* Screen-এর কত অংশ ব্যবহার করবে।

এটিকে **Layout** বা **Reflow** বলা হয়।

### ধাপ ৯: Painting

Layout সম্পন্ন হলে Browser প্রতিটি Element Screen-এ আঁকে (Paint করে)।

যেমন—

* Text
* Image
* Background
* Border
* Shadow

সবকিছু Screen-এ দৃশ্যমান হয়।

### ধাপ ১০: JavaScript Execute করা

যদি HTML-এর সাথে JavaScript যুক্ত থাকে, Browser সেটি Execute করে।

উদাহরণ—

```html id="z5n2k7"
<script src="script.js"></script>
```

JavaScript ব্যবহার করে—

* Button Click
* Form Validation
* Animation
* API Call
* Dynamic Content

ইত্যাদি কাজ করা যায়।

### ধাপ ১১: সম্পূর্ণ Web Page প্রদর্শন

সবশেষে Browser একটি সম্পূর্ণ Web Page ব্যবহারকারীর সামনে প্রদর্শন করে।

ব্যবহারকারী এখন Page-এর সাথে Interaction করতে পারে।

## HTML, CSS এবং JavaScript কীভাবে একসাথে কাজ করে?

| প্রযুক্তি  | কাজ                                              |
| ---------- | ------------------------------------------------ |
| HTML       | Web Page-এর Structure তৈরি করে                   |
| CSS        | Web Page-এর Design ও Style নির্ধারণ করে          |
| JavaScript | Web Page-এ Functionality ও Interactivity যোগ করে |

তিনটি প্রযুক্তি একসাথে কাজ করে একটি পূর্ণাঙ্গ Website তৈরি করে।

## বাস্তব উদাহরণ

ধরুন আপনি একটি বাড়ি তৈরি করছেন।

* **HTML** → বাড়ির ভিত্তি, দেয়াল, দরজা ও কক্ষ তৈরি করে।
* **CSS** → বাড়ির রং, টাইলস, সাজসজ্জা ও সৌন্দর্য বৃদ্ধি করে।
* **JavaScript** → দরজা স্বয়ংক্রিয়ভাবে খোলা, লাইট নিয়ন্ত্রণ, নিরাপত্তা ব্যবস্থা ও অন্যান্য স্মার্ট ফিচার পরিচালনা করে।

অর্থাৎ—

* **HTML** → কী থাকবে।
* **CSS** → দেখতে কেমন হবে।
* **JavaScript** → কীভাবে কাজ করবে।

## HTML কী একাই Website তৈরি করতে পারে?

হ্যাঁ, HTML একাই একটি **Static Web Page** তৈরি করতে পারে।

তবে একটি আধুনিক Website তৈরির জন্য সাধারণত নিচের প্রযুক্তিগুলো একসাথে ব্যবহার করা হয়—

* HTML → Structure
* CSS → Design
* JavaScript → Functionality
* Backend Language → Server-side Logic
* Database → Data সংরক্ষণ

## সারসংক্ষেপ

HTML একটি Web Page-এর ভিত্তি তৈরি করে। ব্যবহারকারী যখন কোনো Website খুলে, তখন Browser Server-এর কাছ থেকে HTML File সংগ্রহ করে, সেটিকে Parse করে DOM তৈরি করে, CSS প্রয়োগ করে, Render Tree তৈরি করে, Layout গণনা করে, Page Paint করে এবং JavaScript চালিয়ে একটি সম্পূর্ণ Web Page প্রদর্শন করে। তাই HTML একা কাজ করে না; Browser, CSS, JavaScript এবং Web Server-এর সাথে সমন্বয় করে একটি পূর্ণাঙ্গ Website ব্যবহারকারীর সামনে উপস্থাপন করে।

HTML (HyperText Markup Language) হলো প্রতিটি Website-এর **ভিত্তি (Foundation)**। একটি Website তৈরির প্রথম ধাপ হলো HTML ব্যবহার করে তার **Structure (কাঠামো)** তৈরি করা। HTML ছাড়া কোনো Web Page-এর মূল গঠন তৈরি করা সম্ভব নয়।

যেমন একটি বাড়ি নির্মাণের আগে তার ভিত্তি, দেয়াল এবং কক্ষ তৈরি করা হয়, ঠিক তেমনি একটি Website তৈরির আগে HTML দিয়ে তার কাঠামো তৈরি করা হয়। এরপর CSS দিয়ে ডিজাইন এবং JavaScript দিয়ে বিভিন্ন কার্যকারিতা (Functionality) যোগ করা হয়।

সহজ ভাষায়—

* **HTML** → Website-এর কাঠামো (Structure)
* **CSS** → Website-এর ডিজাইন (Design)
* **JavaScript** → Website-এর কার্যকারিতা (Functionality)

## Website তৈরিতে HTML-এর প্রধান ভূমিকা

### ১. Website-এর Structure তৈরি করা

HTML-এর সবচেয়ে গুরুত্বপূর্ণ কাজ হলো একটি Website-এর কাঠামো তৈরি করা।

HTML-এর মাধ্যমে Website-এর বিভিন্ন অংশ নির্ধারণ করা হয়, যেমন—

* Header
* Navigation Menu
* Main Content
* Sidebar
* Footer

এই অংশগুলো HTML Tag ব্যবহার করে তৈরি করা হয়।

### ২. Content প্রদর্শন করা

Website-এর সব ধরনের Content HTML-এর মাধ্যমে Browser-এ প্রদর্শিত হয়।

যেমন—

* Heading
* Paragraph
* Image
* Video
* Audio
* Table
* List
* Button
* Link

HTML Browser-কে জানায় কোন Content কীভাবে প্রদর্শিত হবে।

### ৩. বিভিন্ন Web Page সংযুক্ত করা

HTML-এর `<a>` (Anchor) Tag ব্যবহার করে একটি Page-এর সাথে অন্য Page সংযুক্ত করা যায়।

উদাহরণ—

* Home → About
* About → Contact
* Contact → Services

এই Link-গুলোর মাধ্যমে ব্যবহারকারী সহজে Website-এর বিভিন্ন Page-এ যেতে পারে।

### ৪. Form তৈরি করা

Website-এ ব্যবহারকারীর কাছ থেকে তথ্য নেওয়ার জন্য HTML Form ব্যবহার করা হয়।

যেমন—

* Login Form
* Registration Form
* Contact Form
* Search Form
* Feedback Form

HTML Form-এর মাধ্যমে তথ্য সংগ্রহ করা হয়, আর Backend সেই তথ্য প্রক্রিয়াকরণ করে।

### ৫. Multimedia যুক্ত করা

HTML ব্যবহার করে Website-এ বিভিন্ন Multimedia যোগ করা যায়।

যেমন—

* Image
* Audio
* Video

### ৬. Semantic Structure তৈরি করা

HTML5-এর Semantic Tag ব্যবহার করে Website-এর প্রতিটি অংশকে অর্থপূর্ণভাবে ভাগ করা যায়।

গুরুত্বপূর্ণ Semantic Tag—

* `<header>`
* `<nav>`
* `<main>`
* `<section>`
* `<article>`
* `<aside>`
* `<footer>`

এতে Browser, Search Engine এবং Developer সহজে Website-এর Structure বুঝতে পারে।

### ৭. SEO (Search Engine Optimization)-এ সাহায্য করা

সঠিক HTML Structure ব্যবহার করলে Search Engine Website-এর Content আরও ভালোভাবে বুঝতে পারে।

উদাহরণ—

* Heading Tag (`<h1>`–`<h6>`)
* Semantic Tag
* Image-এর `alt` Attribute
* Title Tag
* Meta Tag

এসব SEO উন্নত করতে গুরুত্বপূর্ণ ভূমিকা রাখে।

### ৮. Accessibility উন্নত করা

HTML-এর সঠিক Tag ব্যবহার করলে Website প্রতিবন্ধী (Disabled) ব্যবহারকারীদের জন্যও সহজে ব্যবহারযোগ্য হয়।

উদাহরণ—

* `alt` Attribute
* Label Tag
* Semantic Element
* Form Label

এসব Screen Reader-কে Content বুঝতে সাহায্য করে।

### ৯. Responsive Website-এর ভিত্তি তৈরি করা

Responsive Design মূলত CSS-এর মাধ্যমে করা হলেও, একটি সঠিক HTML Structure Responsive Website তৈরির ভিত্তি হিসেবে কাজ করে।

পরিষ্কার ও অর্থবহ HTML Code বিভিন্ন Screen Size-এ Website মানিয়ে নিতে সাহায্য করে।

### ১০. অন্যান্য Web Technology-এর ভিত্তি তৈরি করা

HTML অন্যান্য Web Technology-এর সাথে একসাথে কাজ করে।

| প্রযুক্তি  | HTML-এর ভূমিকা                                          |

# > 1.11 History of HTML

HTML (HyperText Markup Language)-এর ইতিহাস শুরু হয় **১৯৮৯ সালে**, যখন **Tim Berners-Lee** বিশ্বব্যাপী তথ্য সহজে আদান-প্রদানের জন্য একটি নতুন পদ্ধতির ধারণা দেন। তিনি এমন একটি ব্যবস্থা তৈরি করতে চেয়েছিলেন, যেখানে বিশ্বের যেকোনো স্থান থেকে মানুষ সহজে তথ্য পড়তে, শেয়ার করতে এবং একে অপরের সাথে সংযুক্ত হতে পারে।

এই ধারণা থেকেই **World Wide Web (WWW)** এবং HTML-এর জন্ম হয়। HTML-এর মাধ্যমে বিভিন্ন ডকুমেন্টকে **Hyperlink** ব্যবহার করে একে অপরের সাথে সংযুক্ত করা সম্ভব হয়, যা আজকের আধুনিক Web-এর ভিত্তি।

## HTML তৈরির পেছনের কারণ

১৯৮০-এর দশকের শেষ দিকে বিভিন্ন কম্পিউটারে সংরক্ষিত তথ্য একে অপরের সাথে সহজে সংযুক্ত করা যেত না।

তখন প্রয়োজন ছিল এমন একটি প্রযুক্তির, যা—

* বিভিন্ন কম্পিউটারের তথ্য সংযুক্ত করতে পারবে।
* সহজে পড়া ও শেয়ার করা যাবে।
* Internet-এর মাধ্যমে যেকোনো স্থান থেকে ব্যবহার করা যাবে।
* বিভিন্ন Document-এর মধ্যে Link তৈরি করতে পারবে।

এই সমস্যার সমাধানের জন্য HTML তৈরি করা হয়।

## HTML-এর জন্ম

১৯৮৯ সালে **Tim Berners-Lee** World Wide Web-এর ধারণা প্রকাশ করেন।

পরবর্তীতে **১৯৯০ সালে** তিনি HTML-এর প্রথম সংস্করণ তৈরি করেন এবং একই সাথে আরও কয়েকটি গুরুত্বপূর্ণ প্রযুক্তি তৈরি করেন—

* HTML (HyperText Markup Language)
* HTTP (HyperText Transfer Protocol)
* URL (Uniform Resource Locator)
* প্রথম Web Browser
* প্রথম Web Server

এই প্রযুক্তিগুলোর সমন্বয়েই আধুনিক Web-এর সূচনা হয়।

## প্রথম HTML Document

HTML-এর প্রথম Document ছিল খুবই সাধারণ।

এতে ছিল—

* Heading
* Paragraph
* Hyperlink

এবং কিছু মৌলিক Tag।

তখনকার Web Page-এ কোনো ছবি, ভিডিও, Animation বা আধুনিক ডিজাইন ছিল না।

## HTML-এর বিকাশ

HTML জনপ্রিয় হওয়ার সাথে সাথে নতুন নতুন Feature যুক্ত হতে থাকে।

প্রতিটি নতুন Version-এ আরও উন্নত সুবিধা যোগ করা হয়।

### HTML 1.0 (1993)

* HTML-এর প্রথম প্রকাশিত Version।
* শুধুমাত্র সাধারণ Web Page তৈরি করা যেত।
* খুব অল্প কিছু Tag ছিল।

### HTML 2.0 (1995)

* HTML-এর প্রথম Official Standard।
* Form এবং Input-এর সমর্থন যোগ হয়।
* বিভিন্ন Browser-এর মধ্যে সামঞ্জস্য (Compatibility) বৃদ্ধি পায়।

### HTML 3.2 (1997)

* Table যুক্ত হয়।
* Font Tag যুক্ত হয়।
* আরও উন্নত Layout তৈরি করা সম্ভব হয়।

### HTML 4.01 (1999)

* CSS-এর ব্যবহার আরও গুরুত্বপূর্ণ হয়ে ওঠে।
* JavaScript-এর সমর্থন উন্নত হয়।
* Accessibility এবং Internationalization উন্নত করা হয়।
* Structure এবং Presentation আলাদা করার ধারণা গুরুত্ব পায়।

### XHTML (2000)

HTML-কে XML-এর নিয়ম অনুযায়ী আরও কঠোর (Strict) Syntax-এ ব্যবহার করা শুরু হয়।

যেমন—

* সব Tag Close করতে হতো।
* Lowercase Tag ব্যবহার করতে হতো।
* Attribute-এর মান Quote-এর মধ্যে লিখতে হতো।

### HTML5 (2014)

HTML5 আধুনিক Web Development-এ একটি বড় পরিবর্তন নিয়ে আসে।

HTML5-এ যুক্ত হয়—

* Semantic Tag
* Audio
* Video
* Canvas
* SVG
* Local Storage
* Geolocation
* Drag and Drop
* Offline Support

বর্তমানে HTML5-ই সবচেয়ে বেশি ব্যবহৃত Version।

HTML আর নির্দিষ্ট সময় পরপর নতুন Version (যেমন HTML6) হিসেবে প্রকাশ করা হয় না। বরং HTML5-কে ভিত্তি করে প্রয়োজন অনুযায়ী নিয়মিত নতুন Feature, উন্নতি এবং সংশোধন যোগ করা হয়।

এ কারণে বর্তমান Web Development-এ **HTML5**-ই Standard হিসেবে ব্যবহৃত হয়।

## HTML ইতিহাসের গুরুত্বপূর্ণ ঘটনা

| 1993    | HTML 1.0 প্রকাশিত হয়।                                     |
| 1995    | HTML 2.0 Official Standard হয়।                            |

| 2000    | XHTML প্রকাশিত হয়।                                        |

## HTML-এর ইতিহাস জানা কেন গুরুত্বপূর্ণ?

HTML-এর ইতিহাস জানলে—

* HTML কেন তৈরি হয়েছে তা বোঝা যায়।
* HTML-এর বিবর্তন সম্পর্কে ধারণা পাওয়া যায়।
* পুরোনো এবং নতুন Feature-এর পার্থক্য বোঝা যায়।
* আধুনিক Web Development-এর ভিত্তি সম্পর্কে পরিষ্কার ধারণা তৈরি হয়।
* HTML5 কেন Standard Version তা বোঝা সহজ হয়।

## বাস্তব উদাহরণ

ধরুন, একটি ছোট গ্রামের রাস্তা দিয়ে শুরু হয়েছে একটি যোগাযোগ ব্যবস্থা।

প্রথমে ছিল শুধু সরু পথ। পরে সেই পথ বড় রাস্তা হয়েছে, তারপর মহাসড়ক হয়েছে, এরপর Flyover, Expressway এবং Smart Traffic System যুক্ত হয়েছে।

HTML-এর ইতিহাসও ঠিক এমনই।

* HTML 1.0 → ছোট রাস্তা
* HTML 2.0 → বড় রাস্তা
* HTML 3.2 → আরও উন্নত রাস্তা
* HTML 4.01 → আধুনিক মহাসড়ক
* HTML5 → স্মার্ট, দ্রুত এবং আধুনিক যোগাযোগ ব্যবস্থা

অর্থাৎ, প্রতিটি Version আগের Version-এর তুলনায় আরও উন্নত হয়েছে।

## সারসংক্ষেপ

HTML-এর ইতিহাস শুরু হয় ১৯৮৯ সালে **Tim Berners-Lee**-এর World Wide Web-এর ধারণা থেকে। ১৯৯০ সালে HTML তৈরি হয় এবং এরপর HTML 1.0 থেকে HTML5 পর্যন্ত ধারাবাহিকভাবে উন্নত হয়েছে। বর্তমানে HTML একটি **Living Standard**, যা নিয়মিত উন্নত হচ্ছে এবং আধুনিক Web Development-এর সবচেয়ে গুরুত্বপূর্ণ ভিত্তি হিসেবে ব্যবহৃত হচ্ছে।






# > 1.12 Frontend এর 3টি প্রধান প্রযুক্তি

Frontend Development-এর ৩টি প্রধান প্রযুক্তি হলো—

1. **HTML (HyperText Markup Language)**
2. **CSS (Cascading Style Sheets)**
3. **JavaScript (JS)**

এই তিনটি প্রযুক্তি একসাথে ব্যবহার করে একটি সম্পূর্ণ, সুন্দর এবং ইন্টারেক্টিভ (Interactive) Website তৈরি করা হয়।

সহজভাবে বলতে গেলে—

* **HTML** → Website-এর কাঠামো (Structure) তৈরি করে।
* **CSS** → Website-কে সুন্দর ও আকর্ষণীয় (Design & Styling) করে।
* **JavaScript** → Website-এ কার্যকারিতা (Functionality) ও ইন্টারঅ্যাকশন (Interactivity) যোগ করে।

## Frontend কী?

Frontend হলো Website-এর সেই অংশ, যা ব্যবহারকারী (User) সরাসরি Browser-এ দেখতে এবং ব্যবহার করতে পারে।

উদাহরণ—

* লেখা (Text)
* ছবি (Image)
* Button
* Menu
* Form
* Animation
* Video
* Icon

এসবই Frontend-এর অংশ।

## ১. HTML (HyperText Markup Language)

HTML হলো প্রতিটি Website-এর ভিত্তি (Foundation)।

এটি Website-এর **Structure** তৈরি করে এবং Browser-কে জানায় কোন অংশটি কী ধরনের Content।

HTML-এর মাধ্যমে তৈরি করা যায়—

* Heading
* Paragraph
* Image
* Link
* Table
* List
* Form
* Audio
* Video

উদাহরণ—

```html id="x4m7q9"
<h1>Welcome</h1>
<p>This is my first website.</p>
```

### HTML-এর কাজ

* Website-এর কাঠামো তৈরি করা।
* Content সাজানো।
* বিভিন্ন Page সংযুক্ত করা।
* Form তৈরি করা।
* Multimedia যুক্ত করা।

## ২. CSS (Cascading Style Sheets)

CSS-এর কাজ হলো HTML দিয়ে তৈরি Website-কে সুন্দরভাবে সাজানো (Styling)।

CSS ব্যবহার করে পরিবর্তন করা যায়—

* Text-এর রং
* Font
* Background
* Width
* Height
* Margin
* Padding
* Border
* Layout
* Animation

উদাহরণ—

```css id="r8k2w5"
h1 {
    color: blue;
    font-size: 40px;
}
```

### CSS-এর কাজ

* Design তৈরি করা।
* Responsive Layout তৈরি করা।
* Animation যোগ করা।
* Website-কে Professional দেখানো।

## ৩. JavaScript (JS)

JavaScript হলো Website-এর **Programming Language**।

এটি Website-এ Logic, Functionality এবং Interactivity যোগ করে।

JavaScript ব্যবহার করে—

* Button Click
* Form Validation
* Image Slider
* Calculator
* Game
* Dynamic Content
* API Call
* Popup

ইত্যাদি তৈরি করা যায়।

উদাহরণ—

```javascript id="p6n3h8"
alert("Hello World");
```

### JavaScript-এর কাজ

* User Interaction পরিচালনা করা।
* HTML পরিবর্তন করা।
* CSS পরিবর্তন করা।
* Dynamic Website তৈরি করা।
* Server-এর সাথে Data আদান-প্রদান করা।

## HTML, CSS এবং JavaScript কীভাবে একসাথে কাজ করে?

ধরুন একটি Website-এ একটি Button আছে।

### HTML

Button তৈরি করবে।

```html id="v1t5c7"
<button>Click Me</button>
```

### CSS

Button-এর রং, আকার এবং ডিজাইন নির্ধারণ করবে।

```css id="y9d4m2"
button {
    background: blue;
    color: white;
}
```

### JavaScript

Button-এ Click করলে কী হবে তা নির্ধারণ করবে।

```javascript id="k7b8q1"
button.onclick = function () {
    alert("Button Clicked!");
};
```

অর্থাৎ—

* HTML → Button তৈরি করেছে।
* CSS → Button সুন্দর করেছে।
* JavaScript → Button-কে কাজ করার ক্ষমতা দিয়েছে।

## তিনটি প্রযুক্তির তুলনা

| প্রযুক্তি  | প্রধান কাজ                            | এটি ছাড়া কী হবে?                                                   |
| ---------- | ------------------------------------- | ------------------------------------------------------------------- |
| HTML       | Structure তৈরি করে                    | Website-এর কাঠামো থাকবে না।                                         |
| CSS        | Design ও Styling করে                  | Website থাকবে, কিন্তু দেখতে সাধারণ ও অগোছালো হবে।                   |
| JavaScript | Functionality ও Interactivity যোগ করে | Website দেখা যাবে, কিন্তু ব্যবহারকারীর সাথে গতিশীলভাবে কাজ করবে না। |

## বাস্তব উদাহরণ

ধরুন, আপনি একটি গাড়ি তৈরি করছেন।

* **HTML** → গাড়ির বডি, দরজা, চাকা এবং আসনের কাঠামো তৈরি করে।
* **CSS** → গাড়ির রং, ডিজাইন এবং সৌন্দর্য বৃদ্ধি করে।
* **JavaScript** → ইঞ্জিন, ব্রেক, স্টিয়ারিং, লাইট এবং অন্যান্য কার্যকরী ব্যবস্থা পরিচালনা করে।

অর্থাৎ—

* **HTML** → কী থাকবে।
* **CSS** → দেখতে কেমন হবে।
* **JavaScript** → কীভাবে কাজ করবে।

# > 1.13 HTML শেখার আগে যা জানা দরকার

HTML শেখা শুরু করার আগে কিছু মৌলিক বিষয় সম্পর্কে ধারণা থাকলে শেখার গতি বাড়ে এবং Web Development সহজে বোঝা যায়। তবে একটি গুরুত্বপূর্ণ বিষয় হলো—**HTML শেখার জন্য আগে Programming Language জানা বাধ্যতামূলক নয়।** HTML একেবারে শুরু থেকে শেখা যায়।

এই অধ্যায়ে এমন কিছু বিষয় আলোচনা করা হয়েছে, যা HTML শেখাকে আরও সহজ ও কার্যকর করে তুলবে।

## HTML শেখার জন্য কী কী জানা দরকার?

### ১. কম্পিউটারের মৌলিক ধারণা

প্রথমে কম্পিউটার সম্পর্কে কিছু সাধারণ ধারণা থাকা উচিত।

যেমন—

* Computer কী?
* Hardware ও Software কী?
* File ও Folder কী?
* Operating System কী?
* Keyboard ও Mouse ব্যবহার
* File Create, Rename, Copy, Move এবং Delete করা

এসব জানা থাকলে HTML File পরিচালনা করা সহজ হবে।

### ৪. Web Browser ব্যবহার

HTML File-এর ফলাফল (Output) দেখার জন্য Browser ব্যবহার করা হয়।

* Google Chrome

### ৫. Internet সম্পর্কে প্রাথমিক ধারণা

নিচের বিষয়গুলো সম্পর্কে সাধারণ ধারণা থাকলে ভালো হয়—

* Web Page কী?

* Domain কী?
* Web Browser কী?

এসব বিষয় HTML কোথায় এবং কীভাবে ব্যবহৃত হয়, তা বুঝতে সাহায্য করে।

# > 1.14 Static vs Dynamic Website

Website-কে সাধারণত দুটি প্রধান ভাগে ভাগ করা হয়—

1. **Static Website**
2. **Dynamic Website**

দুই ধরনের Website-ই Browser-এ HTML হিসেবে প্রদর্শিত হয়, তবে **Content কীভাবে তৈরি হয় এবং পরিবর্তন হয়**—সেখানেই তাদের মূল পার্থক্য।

সহজভাবে বলতে গেলে—

* **Static Website** → সবার জন্য একই Content দেখায়।
* **Dynamic Website** → ব্যবহারকারী, সময় বা তথ্য অনুযায়ী Content পরিবর্তন হতে পারে।

## Static Website কী?

Static Website হলো এমন একটি Website, যেখানে প্রতিটি Web Page-এর Content আগে থেকেই HTML File-এ লেখা থাকে। ব্যবহারকারী যখন Website-টি খোলে, তখন Server সেই HTML File সরাসরি Browser-এ পাঠিয়ে দেয়।

অর্থাৎ, ব্যবহারকারী যতবারই Website খুলুক না কেন, একই Page-এর একই Content দেখতে পাবে (যদি Developer File পরিবর্তন না করেন)।

* তৈরি করা সহজ।
* রক্ষণাবেক্ষণ সহজ (ছোট Website-এর ক্ষেত্রে)।

### Static Website-এর উদাহরণ

### Static Website-এর সুবিধা

* দ্রুত Load হয়।

* Hosting খরচ কম।
* তৈরি করা সহজ।
* ছোট Project-এর জন্য উপযুক্ত।

### Static Website-এর অসুবিধা

* Content পরিবর্তন করতে HTML File Edit করতে হয়।
* User অনুযায়ী আলাদা Content দেখানো যায় না।
* Login System তৈরি করা যায় না।
* Database ব্যবহার করা যায় না।
* বড় Website পরিচালনা কঠিন।

## Dynamic Website কী?

Dynamic Website হলো এমন একটি Website, যেখানে Page-এর Content প্রয়োজন অনুযায়ী পরিবর্তিত হয়।

Content তৈরি হতে পারে—

* ব্যবহারকারীর তথ্য অনুযায়ী।
* Database থেকে।
* সময় অনুযায়ী।
* Server-এর Response অনুযায়ী।
* API থেকে প্রাপ্ত তথ্য অনুযায়ী।

অর্থাৎ, একই Page ভিন্ন ভিন্ন ব্যবহারকারীর জন্য ভিন্ন তথ্য প্রদর্শন করতে পারে।

# > 1.15 HTML, CSS & JavaScript Relationship

HTML, CSS এবং JavaScript হলো Web Development-এর তিনটি মৌলিক প্রযুক্তি। এই তিনটি প্রযুক্তি একসাথে কাজ করে একটি সম্পূর্ণ, সুন্দর এবং ইন্টারেক্টিভ (Interactive) Website তৈরি করে।

প্রতিটি প্রযুক্তির আলাদা দায়িত্ব (Responsibility) রয়েছে, কিন্তু তারা একে অপরের পরিপূরক (Complementary)। একটি প্রযুক্তি একা একটি আধুনিক Website তৈরি করতে পারে না।

সহজভাবে বলতে গেলে—

* **HTML** → Website-এর কাঠামো (Structure) তৈরি করে।
* **CSS** → Website-এর ডিজাইন ও সৌন্দর্য (Style & Design) নির্ধারণ করে।
* **JavaScript** → Website-এ কার্যকারিতা (Functionality) ও ইন্টারঅ্যাকশন (Interactivity) যোগ করে।

* কোনটি Paragraph হবে।
* কোথায় Image থাকবে।
* কোথায় Button থাকবে।

* Font
* Background
* Border

* Height
* Margin
* Padding
* Layout
* Animation

এখানে CSS HTML-এর Element-গুলোর চেহারা পরিবর্তন করেছে।

# > 1.16 Frontend vs Backend

Web Development মূলত দুটি প্রধান অংশে বিভক্ত—

1. **Frontend Development**
2. **Backend Development**

এই দুটি অংশ একসাথে কাজ করে একটি সম্পূর্ণ Website বা Web Application তৈরি করে।

সহজ ভাষায়—

* **Frontend** → ব্যবহারকারী যা **দেখে এবং ব্যবহার করে**।
* **Backend** → পর্দার আড়ালে (Behind the Scenes) যা **কাজ করে**।

উদাহরণ হিসেবে, আপনি যখন একটি Website-এ Login করেন—

* Login Form, Button এবং Page-এর Design হলো **Frontend**।
* Username ও Password যাচাই করা, Database থেকে তথ্য আনা এবং Login সফল করা হলো **Backend**।

## Frontend কী?

Frontend হলো Website-এর সেই অংশ, যা ব্যবহারকারী সরাসরি Browser-এ দেখতে এবং ব্যবহার করতে পারে।

এটি **Client-Side** নামেও পরিচিত, কারণ এর বেশিরভাগ কাজ ব্যবহারকারীর Browser-এ সম্পন্ন হয়।

# > 1.17 Server & Database

Server এবং Database হলো আধুনিক Web Development-এর দুটি অত্যন্ত গুরুত্বপূর্ণ অংশ। একটি Website বা Web Application শুধু HTML, CSS এবং JavaScript দিয়ে সম্পূর্ণ হয় না। ব্যবহারকারীর তথ্য সংরক্ষণ, Login System, Online Shopping, Payment, Message এবং অন্যান্য Dynamic Feature পরিচালনা করার জন্য Server এবং Database প্রয়োজন।

সহজ ভাষায়—

* **Server** → Website-এর কাজ পরিচালনা করে এবং Browser-এর অনুরোধ (Request) প্রক্রিয়া করে।
* **Database** → Website-এর তথ্য (Data) সংরক্ষণ করে।

এই দুটি একসাথে কাজ করে একটি Dynamic Website বা Web Application পরিচালনা করে।

## Server কী?

Server হলো একটি **Computer** অথবা **Software**, যা অন্যান্য Computer (Client)-এর অনুরোধ (Request) গ্রহণ করে এবং তার উত্তর (Response) প্রদান করে।

Web Development-এ Server-এর প্রধান কাজ হলো—

* Website Host করা।
* Browser-এর Request গ্রহণ করা।
* Backend Program চালানো।
* Database-এর সাথে যোগাযোগ করা।
* প্রয়োজনীয় Data Browser-এ পাঠানো।

সহজভাবে বলতে গেলে, Server হলো Website-এর **নিয়ন্ত্রণ কেন্দ্র (Control Center)**।

## Server কীভাবে কাজ করে?

যখন একজন ব্যবহারকারী Browser-এ একটি Website খুলে—

1. Browser Server-এ Request পাঠায়।
2. Server Request গ্রহণ করে।
3. প্রয়োজন হলে Database থেকে Data সংগ্রহ করে।
4. HTML, CSS, JavaScript বা JSON আকারে Response পাঠায়।
5. Browser সেই Response ব্যবহার করে Website প্রদর্শন করে।
